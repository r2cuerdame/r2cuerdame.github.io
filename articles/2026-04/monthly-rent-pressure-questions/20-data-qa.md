# 20-data-qa — 월세 체감 압박을 읽는 5개 질문

- 작성 시각: 2026-04-29 22:36 KST
- 상태: `data_qa_pass_for_story` — 최종 수치 확정이 아니라 30-story로 넘길 검증선 확정.
- 결론: 현재 단계에서는 순위·부담률·특정 동네 단정 금지. 독자에게는 “내 계약 후보지를 무엇으로 다시 물어볼 것인가”를 주는 질문형 기사로 전개한다.

## 1. 데이터 단위 고정

| 항목 | 허용 단위 | 차단선 |
|---|---|---|
| 전월세 거래 | 계약월, 보증금, 월세, 전용면적, 자치구/행정동, 주택유형 | 기준월이 없거나 행정 단위가 섞이면 비교 금지 |
| 면적 비교 | 동일 면적대 또는 면적당 월세 보조지표 | 소형/중형/대형을 한 줄 평균으로 섞지 않기 |
| 보증금 전환 | 공개·명시 가능한 연 환산율을 사용한 시나리오 | 환산율 출처 없으면 월세 환산 결론 금지 |
| 소득/생활비 | 공식 통계 또는 출처가 명확한 보조지표 | 개인 부담률처럼 보이는 확정 표현 금지 |
| 생활권 대안 | 통근·생활시설·대체 후보지 질문 | “싸다/비싸다”만으로 추천 금지 |

## 2. 산식 검증선

- 월세 단순 비교: 동일 지역·동일 면적대·동일 기준월끼리만 비교한다.
- 면적 보정: `월세 / 전용면적`은 보조지표로만 쓰고, 실제 체감 부담의 결론으로 쓰지 않는다.
- 보증금 월세화 시나리오: `월세 + (보증금 × 연 환산율 ÷ 12)` 형식은 가능하지만, 환산율 출처와 “시나리오”라는 표기를 붙인다.
- 변화율: 전월 대비보다 전년동월 또는 전분기 비교를 우선한다. 단, 표본 수가 작으면 방향 판단을 보류한다.
- 지역 비교: 자치구 기준 글이면 자치구로 끝내고, 행정동 기준 글이면 행정동으로 끝낸다. 두 단위를 섞어 순위를 만들지 않는다.

## 3. 과장 결론 차단

- 금지 문장: “이 동네는 월세가 감당 불가”, “무조건 피해야 한다”, “앞으로 더 오른다”, “소득 대비 부담률이 확정됐다”.
- 허용 문장: “이 후보지는 보증금·월세·면적·통근 대안을 같은 표에 놓고 다시 물어볼 필요가 있다”.
- 독자 안전장치: 계약 판단을 대체하지 않고, 현장 확인 질문과 비교 순서만 제공한다.

## 4. 30-story로 넘길 5개 질문

1. 같은 월세라도 보증금과 면적을 바꾸면 체감 압박이 달라지는가?
2. 전년동월 또는 전분기 대비 변화가 실제 선택지를 줄이는 수준인가?
3. 비슷한 통근 시간의 대체 생활권이 있는가?
4. 관리비·교통비·생활시설 접근성이 월세 차이를 상쇄하거나 키우는가?
5. 계약 전 현장에서 어떤 질문을 해야 숫자의 빈칸을 줄일 수 있는가?

## 5. 다음 부서 handoff

- 편집부 30-story는 위 5개 질문을 `발견 → 해석 → 반례/한계 → 현장 질문 → 사용법` 구조로 풀어쓴다.
- 그래픽부는 숫자 순위보다 “같은 월세의 다른 체감”, “보증금-월세 시나리오”, “생활권 대체 질문” 같은 설명형 시각자료를 우선한다.
