# 동네 레이더 매거진 운영체계

_Last updated: 2026-05-02T12:00:43+09:00 KST_

## 범위

- 전용 범위: `recuerdame / #naver`의 동네 레이더 매거진 운영.
- 외부 프로젝트, 다른 채널, 제휴형 글, 계정/결제/권한 상태는 이 문서의 운영 보고 범위 밖이다.
- 현재 실행 환경은 GitHub Pages 공개 허브와 로컬 release candidate 저장소다. 실제 공개 발행은 전용 publisher desk(daily/adaptive)만 처리한다.
- Tistory/외부 블로그 발행 루트는 폐기했고, 공개 원본은 GitHub Pages `/radar/`로 통일한다.

## 현재 전략

- 포지셔닝: **공개 매거진/인사이트형 동네 데이터 글**.
- 1차 독자: **서울/수도권에서 3~12개월 안에 이사·전월세·거주지 비교를 해야 하는 25~39세 직장인/1인가구/신혼·예비신혼**.
- 핵심 약속: `어느 동네가 무조건 좋다`가 아니라 `내 상황에서 걸러야 할 리스크와 현장 확인 질문`을 준다.
- 현재 KPI는 즉시 매출이 아니라 신뢰 가능한 공개 포트폴리오와 검색/반복 방문 신호다.
- 공개 cadence는 **매일 1개가 floor**다. dedicated publisher가 타겟 선명도와 품질을 모두 통과한 후보를 GitHub Pages `/radar/`에 내보내며, backlog가 목표 7개를 넘으면 adaptive 발행부가 하루 최대 4개·최소 2시간 간격으로 추가 공개한다.
- 생산 KPI는 파일 수가 아니라 **타겟에 맞는 품질 통과 미발행 큐**다. 너무 쌓지 않고 최소 3개, 목표 7개, 상한 10개만 유지한다.
- 일일 순증 목표는 **0~+1개 품질 통과 후보**다. 3개 미만이면 최대 2개/day까지 보강·완성하고, 3~7개면 새 후보는 하루 1개까지만 만들며 나머지 턴은 기존 후보 품질 보강에 쓴다. 7개 초과면 새 후보 양산을 멈추고 제목/도입/시각자료/반례/SEO/내부링크 품질을 올린다. 10개 초과면 backlog 감축 모드로 보고하고 생산보다 발행-ready 유지·중복 제거·발행부 소화에 집중한다.
- 글은 완성형 공개 글이어야 한다. 독자를 끊어 세우는 티저, 내부 QA 페이지, 판매 페이지처럼 보이면 실패다.
- 7일/168시간은 개별 글의 발행 지연 게이트가 아니라 rolling buffer 추적 메타다.
- Reader QA + 타겟 품질 게이트 + 정적/편집 가드 통과 + hard/soft issue 0건이면 `검사 충분`으로 본다. 충분히 검사된 clean 후보는 추가 숙성/완벽주의로 묶어두지 말고 dedicated publisher가 baseline/adaptive 슬롯으로 바로 선발행한다.

## 품질 KPI

- 본문 길이: 3,200자 미만 발행 차단, 4,500자 미만 보강, 목표 4,500~7,000자.
- 독자용 시각자료: 최소 6개, 목표 8개 이상.
- 기사 구조: 발견 → 해석 → 반례/한계 → 현장 질문 → 사용법.
- 금칙: 공개 본문에 가격/결제/구독/유료/paywall/내부 제작어/쿠팡·파트너스/구매 유도/광고 설정 문구를 넣지 않는다.
- 원칙: 숫자를 나열하지 말고, 독자가 이번 주에 무엇을 확인할지 알려준다.

## 부서 순환

1. 출판관리부 — 후보/보강/blocked/버퍼 상태와 오늘 우선순위.
2. 기획부 — 제목·검색 의도·독자 상황 재검토.
3. 자료수집부 — 출처·기준월·누락 데이터 확인.
4. 데이터검증부 — 산식·단위·과장 결론 검증.
5. 편집부 — 해석·반례·한계·현장 질문·사용법 보강.
6. 그래픽부 — 시각자료 개수와 모바일 가독성 보강.
7. 카피부 — 제목·첫 문단·메타·문장 흐름 정리.
8. 검수부 — Reader QA와 정적 금칙어/안전 가드.
9. 발행부 — 선발행 가능 여부와 blocker 기록. 슬롯이 열리면 `--mode adaptive`로 GitHub Pages 실제 공개까지 수행한다.
10. 성장분석부 — PV/검색/후속편 학습 1개 기록.

## 발행 막힘 처리

- 계정, 로그인, 캡차, 권한, 카테고리, 스킨, 외부 업로드 실패는 전체 회사 정지가 아니다.
- 통과한 글은 `발행대기/보관`으로 keep하고 다음 후보 제작·보강을 계속한다.
- 막힘은 발행부 리포트에 원인과 사람 조치만 기록한다.
