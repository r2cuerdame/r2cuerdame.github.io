# 동네 레이더 KPI Operating Model

_Last updated: 2026-05-02T12:00:43+09:00 KST_

## 현재 단계

- 단계: awareness-first public magazine.
- 즉시 매출 목표: 현재 보류. 공개 신뢰 자산, 검색 유입, 반복 방문, 저장/공유 반응을 먼저 만든다.
- 첫 KPI: **타겟에 맞는 공개 신뢰 자산**. 글 개수 자체가 KPI가 아니다.
- 운영 원칙: `양보다 타겟 적합도`. 매 글은 1차 독자의 실제 이사/계약/동네비교 결정 하나를 도와야 한다.
- 발행 원칙: 각 글의 타겟이 명확하고 품질 게이트를 통과하면 **매일 1개는 기본 최소**다. backlog가 쌓이면 발행부가 adaptive 슬롯으로 추가 공개한다. 단, 기준 미달 글로 빈 슬롯을 억지로 채우지 않는다.
- 생산 제한: 새 글을 늘리기보다 기존 후보가 1차 타겟의 언어·불안·현장 행동에 정확히 꽂히는지 먼저 고친다.

## 전략 게이트

새 후보나 공개 글은 아래 5문항을 통과해야 한다.

1. **누구 글인가:** 서울/수도권에서 3~12개월 안에 이사·전월세·거주지 비교를 해야 하는 개인에게 바로 닿는가?
2. **어떤 결정인가:** 계약 전 확인, 통근 피로, 관리비, 월세 압박, 비 오는 날 집보기처럼 실제 행동/보류 판단이 있는가?
3. **왜 지금 필요한가:** 검색 의도·계절성·생활 이벤트·시장 불안 중 최소 1개가 분명한가?
4. **기존 글과 뭐가 다른가:** 이미 있는 후보와 중복되지 않고 시리즈 내 역할이 있는가?
5. **독자가 들고 나갈 도구가 있는가:** 현장 질문, 비교표, 보류 신호, 설명 문장 중 최소 1개를 제공하는가?

## 타겟 품질 게이트

아래 4개 중 하나라도 약하면 글을 더 쓰지 말고 기존 후보를 되돌려 고친다.

1. **첫 화면 타겟 고정:** 도입 600자 안에 `서울·수도권`, `25~39세`, `3~12개월 안에 전월세/이사/거주지 비교` 맥락이 보여야 한다.
2. **한 사람에게 말하기:** 제목·도입·표·질문이 모두 `지금 집을 계속 볼지/보류할지` 판단하는 개인에게 향해야 한다.
3. **검색어보다 상황:** `OO동`, `월세`, `통근`, `관리비` 같은 키워드를 넣는 데서 끝내지 않고, 방문 전/방문 중/계약 전 행동으로 이어져야 한다.
4. **중복 제거:** 기존 후보와 같은 불안을 다른 제목으로 반복하면 실패다. 새 후보보다 기존 후보의 타겟 선명도·반례·도구성을 높인다.

## 계산 메모

도구 계산 결과:

- dedicated publisher는 타겟과 품질을 모두 통과한 후보가 있으면 매일 1개를 baseline으로 발행한다. adaptive 모드는 ready_unpublished가 목표 7개를 넘을 때 하루 최대 4개·최소 2시간 간격으로 추가 공개하므로, 주간 최대치는 상태에 따라 7개를 넘을 수 있다.
- 미발행 품질후보는 최소 3개, 목표 7개, 상한 10개. 10개 초과 시 새 후보 생성보다 backlog 감축/품질 보강이 우선이다.
- 독자용 시각자료 8개 기준으로 매일 1개 발행 시 주간 56개 visual 검수가 필요하다. 따라서 양산보다 재사용 가능한 시리즈 구조와 검수 품질이 중요하다.

## 현행 KPI

| KPI | 기준 | 현재 적용 |
|---|---:|---|
| 타겟 적합도 | 도입 600자 안에 1차 타겟·결정상황·현장 행동이 보임 | 실패하면 발행 후보 제외 |
| 전략 fit | 전략 게이트 5문항 통과 | 실패하면 새 후보 생성 금지/기획부 환류 |
| publish-ready 후보 | 최소 3개, 목표 7개, 상한 10개 | 10개 초과 시 backlog 감축 모드 |
| 본문 품질 | 4,500~7,000자 | 4,500자 미만 보강 |
| 시각자료 | 8개 이상 | 8개 미만 보강 |
| 공개 금칙어 | 0건 | visible body scan |
| 공개 발행 | 타겟+품질 통과 후보 있으면 baseline 1개 + backlog adaptive 추가 | dedicated publisher desk만, 발행부 턴은 `--mode adaptive` 허용 |

## 미래 옵션

- 방문/검색/반복 방문 데이터가 쌓인 뒤 광고, 리포트, 협업, 데이터 패키지 가능성을 관찰한다.
- 해당 옵션은 현재 공개 본문에 직접 노출하지 않는다.
