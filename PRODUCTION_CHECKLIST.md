# 동네 레이더 Production Checklist

_Last updated: 2026-05-02 12:31 KST_

## Stage checklist

- [ ] 00-topic: 1차 독자(서울/수도권 이사·계약 예정 개인), 검색 의도, 기사 약속, 다음 확인 행동.
- [ ] 10-source: 사용할 공개 출처, 기준월, 단위, 수집 방식, 누락 위험.
- [ ] 20-data-qa: 산식, 지역 단위, 기간 비교, 과장 결론 방지.
- [ ] 30-story: 발견 → 해석 → 반례/한계 → 현장 질문 → 사용법.
- [ ] 40-visual: 최소 6개/목표 8개 이상의 독자용 시각자료 + visual rhythm(상단 루프/요약카드, 중간 callout/카드, 표만 연속 금지).
- [ ] 50-copy: 제목, 첫 문단, 메타 설명, 문장 흐름, 저장/공유 이유.
- [ ] 60-reader-qa: scorecard와 정적 가드 통과.
- [ ] 70-final-html: release candidate final.html 생성, 본문에서 내부어 제거, 렌더링 후 글+표만 보이지 않는지 확인.
- [ ] 70-publish: 발행부 슬롯 확인. GitHub Pages는 dedicated publisher desk(daily/adaptive)만 실제 공개 가능.
- [ ] 80-growth: 검색어, 반복 방문, 후속편 가설 기록.

## 공개 기사 작성 규칙

- 완성형 매거진 글로 작성한다.
- 순위·수치가 필요한 글은 출처와 산식이 확보될 때까지 순위를 확정하지 않는다.
- 본문 안에는 독자가 바로 쓸 질문·비교표·한계·다음 행동이 있어야 한다. 단, 표는 보조자료이며 상단 visual scan/요약카드와 중간 callout/카드로 읽는 리듬을 만든다.
- 제휴/판매/계정/운영 내부어는 공개 본문에 넣지 않는다.
