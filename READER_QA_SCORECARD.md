# 동네 레이더 Reader QA Scorecard

_Last updated: 2026-04-29 21:44 KST_

## Fail-closed gates

| Gate | Pass 기준 | Fail 처리 |
|---|---:|---|
| Target fit | 첫 300자 안에 누가/어떤 결정/무엇을 확인할지 명시 | 없으면 기획부로 되돌림 |
| Visible body length | 4,500자 이상 | 3,200자 미만 hard block, 4,500자 미만 보강 |
| Reader visuals | 8개 이상 | 6개 미만 hard block, 8개 미만 보강 |
| 금칙어/내부어 | visible text 0건 | 즉시 보강/차단 |
| 구조 | 발견→해석→반례/한계→현장 질문→사용법 | 빠진 축 보강 |
| 데이터 안전 | 출처·기준월·단위·산식 caveat 명시 | 불명확하면 순위/강한 결론 금지 |
| 과장 표현 | 확정/보장/필연 표현 없음 | 표현 완화 |

## 기사성 체크

- 1차 독자(서울/수도권 이사·계약 예정 개인)의 상황이 제목/도입에서 바로 보이는가?
- 독자가 숫자를 외우는 대신 판단 순서를 얻는가?
- 좋은 숫자와 나쁜 숫자가 동시에 있는 반례를 보여주는가?
- 동네를 직접 볼 때 물어볼 질문이 있는가?
- 다음 글이나 다음 확인 행동으로 자연스럽게 이어지는가?
- 공개 본문이 내부 산출물이나 판매 페이지처럼 보이지 않는가?

## 점수 기준

- 90점 이상: 선발행 후보. 단, hard/soft issue 0건이어야 한다.
- 80~89점: 보강 후 재검수.
- 79점 이하: 편집부로 되돌림.

현재 자동 가드는 점수보다 hard/soft issue 0건을 우선한다.
